# 简体中文本地化规范

## 目标

本项目的最终运营人员、普通用户和中转站节点运营者都以中文为主，因此前台与后台默认语言统一为简体中文。

本规范用于统一：

- 页面默认语言
- 产品术语翻译
- 时间、日期、数字、货币展示
- 文案语气

## 默认语言

- 公共站与后台默认 `lang="zh-CN"`
- 除非是第三方品牌名、协议名、模型名，否则界面优先使用中文
- 模型名、协议名保留行业常用英文，例如：
  - `GPT 5.4`
  - `OpenAI Responses`
  - `Anthropic Messages`

## 术语约定

| 英文 | 中文约定 | 备注 |
|---|---|---|
| relay | 中转站 | 面向用户展示时优先使用 |
| leaderboard | 榜单 | 公共页面常用词 |
| probe | 探测 / 自检探测 | 面向节点运营者时可写“自检探测” |
| submission | 提交申请 | 用于节点收录流程 |
| sponsor | 赞助位 | 必须与自然排名区分 |
| natural ranking | 自然排名 | 不得与赞助位混淆 |
| methodology | 评分方法 | 也可在正文中补充“Methodology” |
| policy | 收录规则 | 面向普通用户时更易懂 |
| health status | 健康状态 | 公共页面统一表达 |
| degraded | 降级 | 表示服务可达但状态变差 |
| down | 不可用 | 表示当前不可正常服务 |
| paused | 暂停展示 | 表示临时不参与公开排序 |
| unknown | 待确认 | 表示证据不足 |

## 时间与时区

- 默认使用 `zh-CN`
- 默认使用 24 小时制
- 用户可见时间统一展示为北京时间（`Asia/Shanghai`）
- 需要精确展示时，格式优先：
  - `2026/04/17 14:30`
  - `2026年4月17日 14:30`

## 数字与货币

- 百分比统一保留 1 到 2 位小数，避免视觉噪声过大
- 延迟统一使用 `ms`
- 价格默认展示 `USD`，但中文文案需明确单位，例如：
  - `输入价格（USD / 100 万 Tokens）`
  - `输出价格（USD / 100 万 Tokens）`

## 文案语气

- 公共站语气应当清晰、可信、解释性强
- 后台语气应当明确、直接、可执行
- 错误提示优先告诉用户：
  - 出了什么问题
  - 用户下一步该做什么
- 成功提示优先告诉用户：
  - 哪个动作完成了
  - 是否会影响公开页面

## 产品规则表达

- 必须反复强调赞助位与自然排名分离
- 必须避免把探测结果写成“官方认证”
- 必须让节点运营者理解：
  - 自检探测只用于诊断
  - 用户提供的测试 Key 默认不持久化
  - 评分来自观测证据，而非商业合作
