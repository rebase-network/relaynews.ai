# 简体中文本地化规范

## 目标

本项目的最终运营人员、普通用户和大模型API服务站运营者都以中文为主，因此前台与后台默认语言统一为简体中文。

本规范用于统一：

- 页面默认语言
- 产品术语翻译
- 时间、日期、数字、货币展示
- 文案语气

## 默认语言

- 公共站与后台默认 `lang="zh-CN"`
- 除非是第三方品牌名、协议名、模型名，否则界面优先使用中文
- 运维和技术表单里的少数字段保留行业常用英文写法，当前已落地的固定写法包括：
  - `Base URL`
  - `API Key`
  - `测试API Key`
- 模型名、协议名保留行业常用英文，例如：
  - `GPT 5.4`
  - `OpenAI Responses`
  - `Anthropic Messages`

## 术语约定

| 英文 | 中文约定 | 备注 |
|---|---|---|
| relay | 站点 / 大模型API服务站 | 列表、说明、CTA 优先用“站点”；表单字段可写“大模型API服务站名称” |
| leaderboard | 榜单 / 站点榜单 | 公共页面常用词 |
| probe | 测试 / 自助测试 | 面向普通用户统一写“测试”；工具入口可写“自助测试” |
| submission | 提交站点 / 提交站点信息 | 用于站点收录流程 |
| sponsor | 赞助方展示 / 赞助商 | 必须与评测排名区分；首页模块可写“赞助商” |
| natural ranking | 评测排名 | 不得与赞助方展示混淆 |
| methodology | 评测方式 | 导航、正文、帮助文案统一用法 |
| policy | 我们怎么做 | 作为 `/policy` 到 `/methodology#governance` 的兼容入口保留 |
| directory | 站点目录 / 公开目录 | 面向普通用户的收录展示入口 |
| probe page | 站点测试 | 顶部导航、页标题与 CTA 统一用法 |
| health status | 健康状态 | 公共页面统一表达 |
| degraded | 降级 | 表示服务可达但状态变差 |
| down | 不可用 | 表示当前不可正常服务 |
| paused | 已暂停 | 表示站点当前不参与自动测试、公开目录与榜单展示 |
| unknown | 未知 | 状态标签使用“未知”，说明文案可补充样本不足 |

## 当前关键文案

以下文案已经在当前前台 / 后台实现中落地，后续调整时应优先保持一致：

- 顶部导航与桥接入口：
  - `评测方式`
  - `我们怎么做`
  - `站点测试`
- 首页与榜单页：
  - `赞助商`
  - `站点评测排名`
- 站点提交与后台编辑：
  - `Base URL`
  - `测试API Key`
- 状态标签：
  - `健康`
  - `降级`
  - `不可用`
  - `已暂停`
  - `未知`

## 时间与时区

- 默认使用 `zh-CN`
- 默认使用 24 小时制
- 用户可见时间统一展示为北京时间（`Asia/Shanghai`）
- 需要精确展示时，格式优先：
  - `2026/04/17 14:30`
  - `2026年4月17日 14:30`

## 数字与货币

- 百分比统一保留 1 到 2 位小数，避免视觉噪声过大
- 延迟统一使用 `ms`
- 价格默认展示 `USD`，但中文文案需明确单位，例如：
  - `输入价格（USD / 100 万 Tokens）`
  - `输出价格（USD / 100 万 Tokens）`

## 文案语气

- 公共站语气应当清晰、可信、解释性强
- 后台语气应当明确、直接、可执行
- 错误提示优先告诉用户：
  - 出了什么问题
  - 用户下一步该做什么
- 成功提示优先告诉用户：
  - 哪个动作完成了
  - 是否会影响公开页面

## 产品规则表达

- 必须反复强调赞助方展示与评测排名分离
- 必须避免把测试结果写成“官方认证”
- 必须让节点运营者理解：
  - 自助测试只用于诊断
  - `/probe` 中用户提供的 API Key 默认不持久化；`/submit` 中提交的测试 Key 属于审核流程输入，可进入受管凭证记录
  - 评分来自观测证据，而非商业合作
