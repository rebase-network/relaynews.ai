# Public Site Visual Style Review - 2026-04-27

## 审阅说明

- 审阅对象：`https://relaynew.ai/`
- 审阅依据：线上真实页面截图与交互结果，参考 `DESIGN.md` 的暖色、锐角、大标题、强品牌识别方向。
- 覆盖页面：首页、模型目录、模型榜单、Relay 详情、评测方式、提交站点、站点测试、404 与移动端主要页面。
- 本文只记录视觉效果和艺术风格问题，不重复展开功能性问题。

## 整体判断

当前页面已经形成了明确的暖色品牌基调：奶油背景、橙黄渐变、锐角卡片、黑色主按钮和单色数据模块构成了基础识别。但整体视觉仍偏“同一种白卡模板铺满所有页面”，缺少更强的艺术主张和页面间节奏差异。

当前更像一个完成度较高的数据工具 MVP，而不是一个有强记忆点的公开品牌站。后续 UI 调整应避免只改颜色和间距，重点应该是建立更稳定的视觉系统：更有辨识度的 hero、更强的数据表格语言、更明确的赞助展示风格、更可读的移动端排版。

## 主要问题与修改建议

### 1. 视觉识别主要依赖橙色和顶部短条

- 问题：几乎所有模块都使用白色或浅奶油卡片、细边框、顶部橙黄短条，品牌记忆点过度集中在同一个装饰元素上。
- 影响：首页、榜单、详情页、方法页的页面性格相似，浏览时容易产生模板感。
- 建议：保留暖色与锐角，但建立更明确的模块类型。Hero 使用更大面积的暖色块和数据仪表感；榜单使用更强表格结构；方法页使用步骤和权重图形；表单页使用流程式布局。

### 2. 首页移动端视觉叙事顺序错误

- 问题：移动端首屏先看到快速测试表单，后看到主标题和产品说明。
- 影响：用户进入页面时先被表单压住，没有先理解站点定位和价值。
- 建议：移动端首屏应先出现品牌、主张、CTA 和关键指标；快速测试作为第二段高亮工具卡出现。

### 3. 大标题在移动端缺少专门排版策略

- 问题：首页、提交页、长模型 key 在移动端容易出现不自然换行；榜单详情页 H1 占据过高首屏。
- 影响：视觉冲击变成拥挤，降低阅读效率。
- 建议：为移动端独立设置标题尺寸、行高、最大宽度和断行规则。中文标题应减少过密负字距；模型 key 应使用紧凑标签或可控断词。

### 4. 榜单视觉不够像“可信数据产品”

- 问题：桌面榜单已经是表格，但视觉权重仍较轻；移动端榜单卡片像普通内容卡，状态、可用性、延迟和价格之间的层次不够清楚。
- 影响：榜单可信度更多依赖文案，而不是数据结构本身。
- 建议：榜单页强化数据产品语言：更清晰的列对齐、不可用状态分区或降权视觉、低样本提示显性化、价格和延迟使用一致的数字排版。

### 5. Relay 详情页像表格转写，缺少健康解释层

- 问题：Relay 详情页主要是模型健康表格，7 天趋势条没有图例，移动端状态挤在右侧，灰色未知样本不易理解。
- 影响：用户能看到数字，但难以判断这些数字可信程度。
- 建议：增加“样本覆盖”和趋势图例；移动端把状态移到模型名下方，使用单行 badge；灰色状态明确标注为“无样本/未知”。

### 6. 赞助区视觉分离不足

- 问题：首页赞助卡虽然有文字说明，但卡片结构与自然榜单预览接近。
- 影响：扫读时仍可能误以为赞助卡也是自然排名的一部分。
- 建议：赞助区使用独立的视觉语法，例如更深的暖色底、斜角标识、明确 `赞助展示` 标签，并避免自然排名中的排名和分数字段。

### 7. 表单页缺少流程感

- 问题：提交页右侧是常规分组表单，左侧是说明卡，但没有形成“填写信息 -> 自动测试 -> 提交审核”的清晰流程。
- 影响：用户会把提交页理解成后台表单，而不是面向站点运营者的引导流程。
- 建议：保留当前字段结构，但在视觉上加入流程段落、步骤编号或进度式说明。提交按钮区应更像最后确认步骤。

### 8. 方法页信息完整但节奏单一

- 问题：方法页内容充分，但大量模块仍是白卡和列表，缺少视觉锚点。
- 影响：页面可读但记忆点弱，用户不容易快速抓住评分权重、状态定义和赞助分离三个重点。
- 建议：评分构成做成更明显的权重图；状态定义使用状态色和短句；治理部分使用对比式布局突出“会影响排名 / 不会改变排名”。

### 9. 触摸目标与页脚完成度不足

- 问题：移动端 footer 图标触摸区域偏小；短页面 footer 没有贴底时会显得页面中断。
- 影响：移动端可用性和完成度下降。
- 建议：全站 shell 使用纵向 flex，让 footer 在短页面贴底；外链图标保持视觉尺寸但扩大点击热区。

### 10. 字体与字距策略偏西文，对中文长句不稳定

- 问题：全站使用偏几何的西文字体和较强字距控制，中文长标题在小屏幕下容易显得机械或断裂。
- 影响：品牌有科技感，但中文阅读舒适度不稳定。
- 建议：保持 Sora/mono 的技术感用于品牌、模型 key 和数字；中文正文与标题适当放松字距和行高，避免移动端大面积负字距。

## 推荐目标视觉方向

目标不是重做成另一个设计系统，而是在当前暖色锐角基础上增强“公开评测站”的可信数据感：

- 首页：品牌宣言 + 快速入口 + 数据快照，保留暖色大面积视觉。
- 榜单：数据表格优先，状态和样本可信度优先，不可用项明确降权。
- Relay 详情：健康证据板，突出模型覆盖、样本覆盖和趋势解释。
- 方法页：规则说明页，使用权重图和对比式模块降低阅读成本。
- 表单页：面向站点运营者的提交流程，不做后台表单感。
- 移动端：先讲清楚，再让用户操作；标题、chips、状态 badge 都要为小屏单独设计。
