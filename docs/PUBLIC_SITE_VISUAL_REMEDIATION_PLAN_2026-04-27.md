# Public Site Visual Remediation Plan - 2026-04-27

## 目标

将 `docs/PUBLIC_SITE_VISUAL_STYLE_REVIEW_2026-04-27.md` 中的视觉问题转成可执行的视觉整改切片，配合 `docs/PUBLIC_SITE_REMEDIATION_PLAN_2026-04-27.md` 中的 UI/UX 整改一起实施。

## 视觉整改原则

- 不重做设计系统，沿用暖色、锐角、黑色主按钮和强标题方向。
- 先解决影响理解和可信度的视觉问题，再做装饰性 polish。
- 移动端单独设计，不简单压缩桌面布局。
- 榜单和 Relay 详情优先服务数据可信度，不为了品牌感牺牲可读性。
- 赞助展示必须比当前更容易扫读识别，且不能混入自然榜单排序。

## 整改切片

### V0-1. 重塑移动端首页首屏

方案：

- 移动端将 hero 文案、CTA、指标放在快速测试之前。
- 快速测试卡改为第二段工具模块，保持高可见但不压过产品定位。
- 缩小移动端 hero 标题字距和字号，避免中文断裂。

验收标准：

- 390px 宽移动端首屏先看到主标题，而不是表单。
- 首屏包含“查看目录”“开始测试”“提交站点”至少两个核心入口。
- 快速测试仍在首页靠前位置，向下滚动一屏内可见。

### V0-2. 强化榜单的数据可信度表达

方案：

- 过滤不适用的正向徽章，避免不可用站点显示 `低延迟`。
- 对不可用或 0% 可用的行增加降权视觉，不与健康行同等表达。
- 移动端榜单卡片强化状态、可用性、延迟、价格的层次和数字对齐。
- 模型切换条增加横向滚动提示或渐隐遮罩。

验收标准：

- `down` 或 0% 可用行不显示 `低延迟`。
- 不可用行视觉上明显弱于健康行。
- 移动端模型切换能看出可以横向滚动。

### V0-3. 改善 Relay 详情健康证据表达

方案：

- 7 天趋势条增加图例：健康、降级、不可用、无样本。
- 增加样本覆盖说明，例如 `样本覆盖 1/7 天`。
- 移动端状态 badge 放到模型名下方，避免右侧挤压。
- 联系方式为空或为“暂无”时不渲染；URL 或邮箱转为可点击链接。

验收标准：

- Relay 详情页用户能明确区分灰色无样本和绿色健康。
- 低样本时有显性文案，不被折叠为 `+1`。
- URL 联系方式可点击，空联系方式不占位。

### V1-1. 让错误页和短页面看起来完整

方案：

- 404 页面改为静态中文错误页，不自动跳转。
- 页面 shell 使用 flex 布局，让 footer 在短页面贴底。
- Footer 外链图标扩大点击热区。

验收标准：

- 404 页面有明确标题、说明、主 CTA 和次 CTA。
- `/probe`、404 等短页面底部没有 footer 后的大块空白。
- 移动端 footer 外链触摸区接近 44px。

### V1-2. 表单视觉流程化

方案：

- `/probe` 和首页快速测试使用中文字段级校验。
- 提交页保留当前字段，但强化提交流程文案和最后确认区。
- 字段错误信息使用统一颜色、间距和位置。

验收标准：

- 空提交不出现英文浏览器气泡。
- 表单错误可持续显示，并能被自动化测试定位。
- 提交流程仍保持简洁，不引入多步复杂状态。

### V2-1. 提升赞助区独立性

方案：

- 赞助区使用独立背景、边框、标签和文案结构。
- 赞助卡不使用自然排名的分数和排名符号。
- 保留“独立展示，不参与榜单排序”文案。

验收标准：

- 首页扫读时赞助区与模型榜单预览明显不同。
- 赞助展示不会被误判为榜单排名。

### V2-2. 调整页面模块节奏

方案：

- 首页保留强 hero，榜单预览减少卡片重复感。
- 方法页用权重图、对比模块和状态定义增强阅读节奏。
- 表单页、测试页保持工具感，不再像后台表单。

验收标准：

- 首页、榜单页、方法页、表单页的模块形态有可见差异。
- 不影响现有信息完整性和中文解释性。

## 不在本轮做的事项

- 不更换整体技术栈。
- 不引入新基础设施。
- 不恢复 Relay 详情中的独立价格历史和事故时间线模块。
- 不改 admin 站点视觉。
